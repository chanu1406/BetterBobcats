# Systems / Infrastructure Engineering Career Path

## üéØ Overview

The **Systems / Infrastructure Engineering** career path prepares students for roles focused on designing, building, and maintaining large-scale infrastructure systems. This specialization emphasizes understanding operating systems, distributed systems, computer architecture, networking, and cloud computing.

### Target Roles
- **Systems Engineer**: Design and maintain large-scale infrastructure systems
- **Infrastructure Engineer**: Build and manage cloud and on-premises infrastructure
- **Site Reliability Engineer (SRE)**: Ensure system reliability, performance, and scalability
- **Platform Engineer**: Build and maintain developer platforms and tooling
- **DevOps Engineer**: Automate infrastructure and deployment pipelines
- **Cloud Engineer**: Design and manage cloud infrastructure (AWS, GCP, Azure)
- **Network Engineer**: Design and maintain network infrastructure
- **Performance Engineer**: Optimize system performance and resource utilization

### Why Choose Systems / Infrastructure Engineering?
- **High Demand**: Critical need for engineers who can build and scale infrastructure
- **High Impact**: Infrastructure decisions affect millions of users
- **Competitive Salaries**: Infrastructure roles command premium compensation
- **Technical Depth**: Deep understanding of how systems work at all levels
- **Scale**: Work on systems that serve millions of users globally
- **Variety**: Work across hardware, software, networking, and cloud technologies

## üìö Course Recommendations by Tier

The Systems / Infrastructure Engineering curriculum is organized into 5 tiers based on importance and impact on career readiness.

### üü¢ **TIER 1: NON-NEGOTIABLE CORE (SYSTEMS ENGINEERING)** (6 courses)

If a student does not take most of these, they are not systems-ready. These courses form the foundational core of systems engineering.

| Course | Code | Why It's Critical |
|--------|------|-------------------|
| **Operating Systems** | CSE 150 | Processes, memory, concurrency - essential for understanding how systems work |
| **Computer Networks** | CSE 160 | TCP/IP, routing, real infrastructure - fundamental networking knowledge |
| **Distributed Software Systems** | CSE 168 | Scalability, fault tolerance - critical for building reliable systems |
| **Computer Architecture** | CSE 140 | Hardware-software interface - understanding system optimization |
| **Parallel Computing** | CSE 179 | Performance, multithreading - essential for modern systems |
| **Computer Organization & Assembly** | CSE 031 | Low-level execution - understanding system internals |

**Priority**: These courses provide the foundational knowledge that all systems engineers need, regardless of specialization.

**Outcome**: After completing Tier 1, students will understand:
- How operating systems manage processes, memory, and concurrency
- Network protocols and infrastructure design
- Distributed system architectures and fault tolerance
- Computer architecture and performance optimization
- Parallel computing and multithreading
- Low-level system execution and hardware interaction

## üéì Academic Timeline

### Freshman Year
- Focus on foundational CS courses
- Take CSE 031 (Computer Organization & Assembly)
- Build programming skills

### Sophomore Year
- Take CSE 150 (Operating Systems)
- Take CSE 160 (Computer Networks)
- Start learning Linux and command-line tools
- Build personal projects

### Junior Year
- Take CSE 168 (Distributed Software Systems)
- Take CSE 140 (Computer Architecture)
- Take CSE 179 (Parallel Computing)
- Start Tier 2 courses (EECS courses)
- Get internships in infrastructure/DevOps roles

### Senior Year
- Complete remaining Tier 1 and Tier 2 courses
- Take Tier 3 advanced courses
- Consider Tier 4 (ME) courses if interested in cyber-physical systems
- Complete capstone project related to infrastructure
- Prepare for full-time roles

## üöÄ Getting Started

1. **Master the Fundamentals**: Complete Tier 1 courses (CSE 150, 160, 168, 140, 179, 031)
2. **Build Projects**: Create infrastructure projects using cloud platforms
3. **Get Hands-On Experience**: Set up your own servers, deploy applications, automate tasks
4. **Learn Tools**: Get comfortable with Docker, Kubernetes, Terraform, and cloud platforms
5. **Contribute to Open Source**: Contribute to infrastructure-related open source projects
6. **Get Internships**: Apply for infrastructure/DevOps internships early
7. **Network**: Attend meetups, conferences, and connect with professionals in the field

## üìù Notes

- **Tier 1 courses are non-negotiable** - take most of these to be systems-ready
- **Tier 2 courses distinguish UCM systems engineers** - these EE courses provide deep hardware knowledge
- **Tier 3 courses are for advanced roles** - focus on these for FAANG-level positions
- **Tier 4 courses are optional** - valuable for cyber-physical systems and robotics
- **Tier 5 courses are optional but powerful** - security and reliability knowledge is always valuable

This career path emphasizes **depth over breadth** - understanding systems at a fundamental level is more valuable than knowing many surface-level tools.
